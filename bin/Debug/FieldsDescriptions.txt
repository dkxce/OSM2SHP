-- Адресные поля:

Адресные теги начинаются с `addr:`
Для сокращения наименования адресный полей `addr:` заменяется на `_`, таким образом:
`addr:flats` --> `_FLATS`
`addr:housenumber` --> `_HOUSENUMBA`
`addr:housename` --> `_HOUSENAME`
`addr:street` --> `_STREET`
`addr:full` --> `_FULL`
Полный список полей опеределяется адресным селектором и задается в файлах .json
в подкаталоге `json_addr`.

-- Дополнительные поля:

Дополнительные поля формируются на основе одноименных тегов, например:
тег `name` будет является полем `NAME`
тег `name:ru` будет являться полем `NAME_RU`
тег `amenity` будет являться полем `AMENITY`
тег `shop` будет являться полем `SHOP`
Максимальная длина наименования поля - 10 символов.
Если имя тега длиннее 10 символов, имя поля будет урезано и в конец добавлен
порядковый суффикс для урезанных полей (от A до Z)
Например: 'addr:housenumber` --> `ADDR_HOUSEA`

Формат дополнительных полей может включать длину поля:
name
name[200]
name:ru[200]
Длина поля по умолчанию 200 символов.
Все дополнительные поля имеют тип 'C' - строка

Если какое-то дополнительное поле будет иметь одинаковый заголовок (например `user` и `USER`) следует
переименовать поля полсе формирования выходного файла в редакторе на несовпадающие значения, иначе
некоторые программы не смогут открыть файл DBF
